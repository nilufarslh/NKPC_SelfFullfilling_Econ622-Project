API Reference

Docstrings for every exported symbol.

Configuration

SelfFulfillingNKPC.ModelParams — Type
ModelParams{T}

All structural and policy parameters of the model. Mutable so that unpack can overwrite the estimated fields while the rest stay at their calibrated values. Parametric in T so that ForwardDiff duals propagate through the entire simulation.

source
SelfFulfillingNKPC.ParamSpec — Type
ParamSpec(name, theta0, lb, ub)

Estimation spec for one parameter: its symbol, starting value, and box bounds. A vector of these lives on EstimationConfig.estimated.

source
SelfFulfillingNKPC.EstimationConfig — Type
EstimationConfig

Everything that defines one estimation case: fixed vs estimated parameters, learning settings, CB policy bounds, burn-in and seed, which moment set to match, and how to compute standard errors. Build one via case_non_iid or case_non_iid_free_phi.

source
SelfFulfillingNKPC.case_non_iid — Function
case_non_iid(; intervention) -> EstimationConfig

Build the standard non-iid case: seven estimated parameters (κ, the two AR(1) persistences and volatilities, and the two measurement-error volatilities). intervention=false freezes the Taylor coefficient at φ_π(κ̂₀) (Case 1); intervention=true re-optimises it every period as the belief updates (Case 2).

source
SelfFulfillingNKPC.case_non_iid_free_phi — Function
case_non_iid_free_phi(; intervention=true) -> EstimationConfig

Variant that treats φ_π and φ_y as free parameters rather than solving the CB's inner policy problem. The objective is globally smooth in θ, so this is the clean path for the AD-vs-finite-difference gradient test.

source
SelfFulfillingNKPC.param_names — Function
param_names(cfg) -> Vector{Symbol}

Names of the estimated parameters in cfg, in vector order.

source

theta0_vec(cfg) -> Vector{Float64}

Starting values for the estimated parameters, in vector order.

SelfFulfillingNKPC.lower_bounds — Function
lower_bounds(cfg) -> Vector{Float64}

Lower box bounds for the estimated parameters.

source
SelfFulfillingNKPC.upper_bounds — Function
upper_bounds(cfg) -> Vector{Float64}

Upper box bounds for the estimated parameters.

source
SelfFulfillingNKPC.unpack — Function
unpack(theta, cfg) -> ModelParams{T}

Turn an estimated parameter vector into a fully-populated ModelParams, preserving the element type T so ForwardDiff duals flow through end-to-end. Fixed fields are taken from cfg.fixed; estimated fields are overwritten from theta in the order given by param_names(cfg).

source

Data

SelfFulfillingNKPC.TargetData — Type
TargetData

Loaded data: a T × 3 matrix X = [π y r] (demeaned, optionally annualised), the sample length T, and the column names. Returned by load_targets.

source
SelfFulfillingNKPC.load_targets — Function
load_targets(path; annualize=false, demean=true) -> TargetData

Read quarterly US data from CSV. Expected columns: date, pi, y, r. The default is NOT to annualise: the FRED/BHP series stored in data/targets.csv are already in annual-percent units (π is the year-over-year CPI inflation rate in %, r is the annualised Federal-Funds rate in %). Set annualize=true only if the input series are quarterly rates. Demeaning subtracts the column mean.

source

Model primitives

SelfFulfillingNKPC.taylor_rule — Function
taylor_rule(π, y, u, φ_π, φ_y) -> r

Evaluate the Taylor rule r = φ_π π + φ_y y + u.

source
SelfFulfillingNKPC.uncond_var — Function
uncond_var(σ, ρ)

Unconditional variance σ² / (1 - ρ²) of an AR(1) with innovation std σ and persistence ρ. ρ is clamped into (-0.9999, 0.9999) so the result stays finite near the unit root.

source
SelfFulfillingNKPC.policy_numeric — Function
policy_numeric(k̂, p, cfg) -> φ_π

The central bank's best-response Taylor coefficient on inflation at belief k̂, found by 1-D Brent search over [cfg.phi_min, cfg.phi_max]. The CB's loss is the subjective Var(π) + λ_y Var(y) computed on the four-shock system (demand, cost-push, and the two measurement errors); θ_r is omitted from the bank's subjective model.

source
SelfFulfillingNKPC.compute_policy — Function
compute_policy(k̂, p, cfg; itp=nothing) -> (φ_π, φ_y)

Return the policy coefficients to apply at belief k̂. Uses the precomputed interpolant itp if given, otherwise falls back to policy_numeric. If cfg.free_phi is set, the free φ_π, φ_y in p are returned directly.

source

Policy interpolant

SelfFulfillingNKPC.PolicyInterp — Type
PolicyInterp

Linear interpolant for φ_π(κ̂): the belief grid and the interpolation object. Built once per SMM objective evaluation so that ForwardDiff can flow through the policy map instead of through Brent's non-differentiable minimiser.

source
SelfFulfillingNKPC.build_policy_interp — Function
build_policy_interp(p, cfg; n_grid=31) -> PolicyInterp

Solve the CB's Brent problem at n_grid equally-spaced beliefs in cfg.k_bounds and wrap the result in a linear interpolant with linear extrapolation.

source
SelfFulfillingNKPC.evaluate_policy_itp — Function
evaluate_policy_itp(itp, k̂, cfg) -> (φ_π, φ_y)

Evaluate the interpolant at belief k̂, clamp φ_π to [phi_min, phi_max], and return alongside the fixed φ_y from cfg.fixed.

source

Simulation

SelfFulfillingNKPC.SimulationResult — Type
SimulationResult{T}

One simulated path: the post-burn-in observable matrix X = [π y r], the three series, and the full-length paths of the belief κ̂ and the applied policy coefficients φ_π, φ_y.

source
SelfFulfillingNKPC.simulate — Function
simulate(theta, cfg, T_obs; inn_d, inn_v, inn_my, inn_mpi, policy_itp)

Persistent RF simulation with decomposed measurement errors. Innovations (N(0,1)) are held fixed; AR(1) processes constructed with current σ,ρ. ut = φπ(t) × mπt + φy × my_t composed each period with current policy.

source

Objective and estimation

SelfFulfillingNKPC.smm_objective — Function
smm_objective(theta, cfg, m_data, W, T_obs; inn_d, inn_v, inn_my, inn_mpi, policy_itp=nothing)

The SMM criterion (m(θ) - m̂)' W (m(θ) - m̂). Innovations are held fixed across evaluations so the objective is a deterministic function of θ; that is what lets ForwardDiff produce exact gradients. If the simulated moments go non-finite, a large quadratic penalty in θ - θ₀ is returned so the outer optimiser can still descend.

source
SelfFulfillingNKPC.estimate_smm — Function
estimate_smm(cfg, m_data, W, T_obs; inn_d, inn_v, inn_my, inn_mpi,
             n_starts=cfg.n_starts, verbose=true, warm_start=nothing,
             logfile=nothing) -> OptimResult

Run multi-start Fminbox(L-BFGS) with ForwardDiff gradients on the SMM objective. Starts are generated by multiplying theta0_vec(cfg) by a fixed sequence of scalars and clamping into the box; this avoids the degenerate κ ≈ 0 basin that low-discrepancy starts on the full box kept finding. Each start has a two-minute time cap. Returns the start with the lowest objective.

source
SelfFulfillingNKPC.OptimResult — Type
OptimResult

Best estimate from one call to estimate_smm: the parameter vector, objective value, convergence flag, number of starts actually attempted, and a stage counter (1 for the single-stage pipeline used in the headline results).

source

Inference and reporting

SelfFulfillingNKPC.CaseResult — Type
CaseResult

Complete output container for one estimation case.

source
SelfFulfillingNKPC.bootstrap_se — Function
bootstrap_se(theta_hat, cfg, m_data, W, T_obs; B=cfg.bootstrap_reps,
             logfile=nothing) -> Vector{Float64}

Nonparametric bootstrap standard errors. B independent innovation paths are drawn at theta_hat, each is re-estimated with a single L-BFGS restart warm-started at theta_hat, and the sample standard deviation of the resulting parameter draws is returned. Runs in parallel across Threads.nthreads() threads.

source
SelfFulfillingNKPC.print_results — Function
print_results(cr)

Print a formatted parameter table (estimate and bootstrap SE) followed by the moment-fit table (data, simulated, difference) to stdout.

source
SelfFulfillingNKPC.save_results — Function
save_results(cr, dir)

Write two CSVs into dir (creating it if missing): params.csv (parameter, estimate, SE) and moments.csv (moment, data, simulated, difference).
